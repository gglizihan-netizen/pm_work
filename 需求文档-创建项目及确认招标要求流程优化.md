# 需求文档：创建项目及确认招标要求流程优化

## 一、需求概述

| 项目 | 内容 |
|------|------|
| 需求名称 | 创建项目及确认招标要求流程优化 |
| 需求来源 | 客户反馈 |
| 优先级 | 高 |
| 影响范围 | 创建项目、确认招标要求 |

---

## 二、需求背景

目前第一步「创建项目」上传文件后会对文件进行解析并展示解析结果，但此时用户无法对解析信息进行编辑；点击「下一步」进入第二步「确认招标要求」页面后才能对信息进行编辑。

**存在问题：** 用户在第一步看到解析的信息后会下意识地进行检查和编辑，但此时无法编辑，导致用户产生困惑，引发较多客户咨询询问为何无法编辑。

---

## 三、需求目标

1. 解决用户在第一步无法编辑解析信息的困惑，优化用户体验
2. 新增工程图纸解析功能，扩展产品能力

---

## 四、需求详细说明

### 4.1 第一步「创建项目」改造

#### 4.1.1 改造内容

**调整文件解析时机：** 将原「上传后立即解析」调整为「仅上传，不解析」，解析环节移至第二步进行。

**新增工程图纸上传入口：** 在原有「上传招标文件」「上传清单文件」基础上，新增「上传工程图纸」入口，支持工程图纸的解析功能。

#### 4.1.2 页面结构

改造后的第一步页面包含以下内容：

| 模块 | 内容说明 |
|------|----------|
| 上传招标文件 | 支持PDF、Word及各地区zf格式，上传成功后显示文件名称和上传状态 |
| 上传清单文件 | 支持PDF、Excel格式，上传成功后显示文件名称和上传状态 |
| 上传工程图纸 | **【新增】** 支持上传工程图纸文件，上传成功后显示文件名称和上传状态 |
| 格式设置 | 生成表格、图文并茂、正文主语、布局方式、正文字体/页面设置等 |
| 页数设置 | 设置方案页数（5~5000页） |

#### 4.1.3 交互说明

1. 用户上传文件后，页面仅显示文件名称和上传状态（如"上传成功"），不进行解析
2. 用户完成文件上传和设置后，点击「下一步」进入第二步

---

### 4.2 第二步「解析招标信息」改造

#### 4.2.1 页面标题

建议将原「确认招标要求」更名为「解析招标信息」，更准确描述该步骤的核心功能。

#### 4.2.2 页面布局

页面采用顶-中-底三段式布局：

```
┌─────────────────────────────────────────────────────────────┐
│  顶部：步骤条（保持不变）                                      │
├─────────────────────────────────────────────────────────────┤
│  中部                                                         │
│  ┌─────────────────────┬───────────────────────────────────┐│
│  │                     │  项目信息展示    [解析状态]         ││
│  │                     │  ├─ 项目名称                      ││
│  │   招标文件展示区     │  ├─ 开标时间                      ││
│  │   （保持不变）       │  ├─ 招标要求                      ││
│  │                     │  ├─ 技术评分标准                  ││
│  │                     │  ├─ 投标文件组成目录              ││
│  │                     │  ├─ 项目清单                      ││
│  │                     │  └─ 工程图纸                      ││
│  │                     ├───────────────────────────────────┤│
│  │                     │  其他补充信息                      ││
│  │                     │  ├─ 目录生成方式选择（条件显示）    ││
│  │                     │  ├─ 补充清单文件提示（条件显示）    ││
│  │                     │  └─ 补充图纸文件提示（条件显示）    ││
│  └─────────────────────┴───────────────────────────────────┘│
├─────────────────────────────────────────────────────────────┤
│  底部：「上一步」（置灰/可点击）  「下一步」（置灰/可点击）      │
└─────────────────────────────────────────────────────────────┘
```

#### 4.2.3 上块：项目信息展示

**功能说明：**
进入该页面后，系统自动调用解析接口，将AI提取的信息以流式输出的方式展示。流式输出即模型逐字输出解析内容，呈现打字机效果。

**提取信息模块：**

| 序号 | 信息模块 | 说明 |
|------|----------|------|
| 1 | 项目名称 | 文本类型，原位编辑 |
| 2 | 开标时间 | 时间类型，使用时间选择器编辑 |
| 3 | 招标要求 | 长文本类型，原位编辑 |
| 4 | 技术评分标准 | 长文本类型，原位编辑 |
| 5 | 投标文件组成目录 | 长文本类型，原位编辑 |
| 6 | 项目清单 | 长文本类型，原位编辑 |
| 7 | 工程图纸 | 长文本类型，原位编辑 |

**解析状态提示：**
- 流式输出过程中：在「项目信息展示」标题旁显示「解析中」状态
- 流式输出结束后：在「项目信息展示」标题旁显示「解析完成」状态

**交互规则：**

| 阶段 | 用户操作 | 按钮状态 | 导航限制 |
|------|----------|----------|----------|
| 流式输出过程中 | 只能观看，不可编辑 | 「下一步」置灰 | 不可点击其他导航离开页面 |
| 流式输出结束后 | 可对各信息模块进行编辑 | 「下一步」可点击 | 可自由导航 |

**编辑说明：**
- 各信息模块支持原位直接编辑，无需弹出编辑弹窗
- 编辑操作支持：删除文字、增加文字
- 项目名称、招标要求等文本类信息：使用文本框编辑
- 开标时间：使用时间选择器编辑
- 项目清单、工程图纸：以长字符串形式展示，支持文本编辑

**异常情况：**
- 某模块解析结果为空时，显示「文件未提取到相关内容」

---

#### 4.2.4 下块：其他补充信息

**功能说明：**
该区域用于展示系统提示信息，包括目录生成方式选择提示、补充文件提示等。

**提示内容及触发条件：**

| 提示类型 | 触发条件 | 提示内容 | 操作入口 |
|----------|----------|----------|----------|
| 目录生成方式选择 | 投标文件组成目录及技术评分标准均有解析内容 | 检测到文件中同时提供技术评分标准及投标文件组成目录，请选择目录生成方式 | 三选一单选框 |
| 补充清单文件提示 | 第一步未上传清单文件 | 当前未上传清单文件，建议补充上传 | 「上传」按钮 |
| 补充图纸文件提示 | 第一步未上传工程图纸 | 当前未上传图纸文件，建议补充上传 | 「上传」按钮 |

**目录生成方式选择：**

当满足触发条件时，在流式输出结束后立即展示目录生成方式选择。

| 选项 | 说明 |
|------|------|
| 融合投标文件组成目录/评分标准 | **默认选中** |
| 仅按评分标准生成 | - |
| 仅按投标文件组成目录生成 | - |

---

#### 4.2.5 补充上传功能

**入口位置：** 「其他补充信息」区域内的对应提示信息处

**交互流程：**

1. 用户点击「上传」按钮，弹出文件选择器
2. 用户选择文件后，提示信息区域展示文件名称
3. 用户点击「提交」按钮触发解析（同时提供「忽略」按钮）
4. 点击「提交」后：
   - 页面自动定位至「项目信息展示」区域的对应信息模块
   - 提示信息变更为「文件解析中」
   - 「下一步」按钮置灰
   - 接口返回结果后，流式输出解析内容
5. 解析完成后：
   - 提示信息变更为「已解析完成」
   - 「下一步」按钮解禁

**按钮说明：**

| 按钮 | 功能说明 |
|------|----------|
| 提交 | 触发文件解析，流式输出解析结果 |
| 忽略 | 关闭提示信息，不进行补充上传 |

---

#### 4.2.6 解析失败处理

**触发场景：** AI解析接口调用失败

**处理方式：**
1. 弹窗提示用户解析失败
2. 弹窗中提供「重新解析」按钮
3. 点击「重新解析」后，重新调用解析接口解析所有内容

## 五、验收标准

### 5.1 第一步验收标准

| 序号 | 验收点 | 预期结果 |
|------|--------|----------|
| 1 | 上传招标文件 | 上传成功后仅显示文件名称和上传状态，不进行解析 |
| 2 | 上传清单文件 | 上传成功后仅显示文件名称和上传状态，不进行解析 |
| 3 | 上传工程图纸 | 【新增功能】上传成功后仅显示文件名称和上传状态，不进行解析 |
| 4 | 点击「下一步」 | 正常进入第二步，并自动触发解析 |

### 5.2 第二步验收标准

| 序号 | 验收点 | 预期结果 |
|------|--------|----------|
| 1 | 进入页面 | 自动调用解析接口，开始流式输出 |
| 2 | 流式输出展示 | 多个信息模块同时流式输出，呈现打字机效果 |
| 3 | 解析状态提示-进行中 | 「项目信息展示」标题旁显示「解析中」 |
| 4 | 解析状态提示-完成 | 「项目信息展示」标题旁显示「解析完成」 |
| 5 | 流式输出期间-按钮状态 | 「下一步」按钮置灰，不可点击 |
| 6 | 流式输出期间-导航限制 | 用户无法点击其他导航离开页面 |
| 7 | 流式输出结束后-可编辑 | 用户可对各信息模块进行原位编辑 |
| 8 | 流式输出结束后-按钮状态 | 「下一步」按钮可点击 |
| 9 | 模块解析结果为空 | 显示「文件未提取到相关内容」 |
| 10 | 目录生成方式选择 | 技术评分标准+投标文件组成目录均有内容时展示，默认选中第一项 |
| 11 | 补充清单文件提示 | 未上传清单文件时展示提示，点击上传可补充 |
| 12 | 补充图纸文件提示 | 未上传工程图纸时展示提示，点击上传可补充 |
| 13 | 补充上传-提交 | 点击后自动定位至对应模块，流式输出解析结果 |
| 14 | 补充上传-忽略 | 关闭提示信息，不进行补充上传 |
| 15 | 解析失败 | 弹窗提示失败，点击「重新解析」可重新调用接口 |
| 16 | 开标时间编辑 | 使用时间选择器进行编辑 |
| 17 | 文本类信息编辑 | 支持删除文字、增加文字 |

## 六、风险评估

| 风险项 | 风险等级 | 应对措施 |
|--------|----------|----------|
| 用户未上传招标文件直接进入第二步 | 中 | 第一步校验招标文件必传，未上传时「下一步」按钮置灰 |
| 流式输出时间过长导致用户等待 | 低 | 页面展示loading动画，提升等待体验 |
| 补充上传文件格式不支持 | 低 | 上传时校验文件格式，不支持的格式提示用户 |
