## 一、需求背景提出

### 1.1 当前问题

目前第一步「创建项目」上传文件后会对文件进行解析，并展示出来解析的信息，但是此时不可编辑；点击「下一步」进入「确认招标要求」页面后才能对信息进行编辑。

**产生的用户困惑：**
- 用户在第一步看到解析的信息后就会下意识地去检查编辑
- 但此时无法进行编辑，对用户造成了很多编辑上的困惑
- 造成很多客户过来询问为什么不能编辑

### 1.2 需求描述

**核心改动：**
- 第一步「创建项目」步骤，将原来的"上传并解析"，改为"仅上传"
- 将"解析环节"放至第二步「确认招标要求」环节

### 1.3 页面构思（第二步改造）

此页面用于流式输出并展示AI提取出的信息（在第一步创建项目时已经上传了招标文件，进入此页面后，会直接流式输出解析的结果）。

**提取信息包括以下几部分：**
1. 项目名称
2. 开标时间
3. 招标要求
4. 技术评分标准
5. 投标文件组成目录
6. 项目清单
7. 工程图纸

**关键交互说明：**
- （1）流式输出即模型一个字一个字地将解析的内容输出出来，类似于打字机效果
- （2）流式输出过程中不可编辑，流式输出结束后各信息支持编辑
- （3）当第一步没有上传清单文件及工程图纸时，需要进行提示，并提供补充上传这两种文件的入口
- （4）当解析出来的投标文件组成目录及技术评分标准均有内容时，需要提供目录生成方式选择功能，提供三种选择：
  - 融合投标文件组成目录/评分标准
  - 仅按评分标准生成
  - 仅按投标文件组成目录生成

## 六、讨论结论

### 6.1 第一步「创建项目」改造要点

1. **新增工程图纸上传入口**
2. **仅上传不解析**，上传后只显示文件名称和状态
3. 保留格式设置和页数设置
4. 完成设置后点击「下一步」进入第二步

### 6.2 第二步「解析招标信息」改造要点

1. **页面标题建议：** "解析招标信息"（替代原"确认招标要求"）

2. **页面布局：**
   - 顶部：步骤条（创建项目 → 解析招标信息 → 生成正文）
   - 中部左侧：招标文件预览区（40%宽度）
   - 中部右侧：
     - 上块：项目信息展示（7个模块竖向排列）
     - 下块：其他补充信息（系统提示区域）
   - 底部：「上一步」「下一步」按钮

3. **流式输出机制：**
   - 进入页面自动触发解析
   - 7个模块同时流式输出（打字机效果）
   - 输出期间「下一步」置灰，不可离开页面
   - 输出结束后可编辑，「下一步」解禁

4. **信息模块编辑：**
   - 项目名称：文本框
   - 开标时间：时间选择器
   - 其他：多行文本框
   - 均支持原位直接编辑

5. **系统提示区域：**
   - 目录生成方式选择（条件显示，默认选中"融合"）
   - 补充清单文件提示（条件显示）
   - 补充图纸文件提示（条件显示）

6. **补充上传流程：**
   - 点击上传 → 选择文件 → 点击提交 → 自动定位 → 流式输出
   - 同时提供"忽略"按钮

7. **异常处理：**
   - 解析失败：弹窗提示 + "重新解析"按钮
   - 模块为空：显示"文件未提取到相关内容"
