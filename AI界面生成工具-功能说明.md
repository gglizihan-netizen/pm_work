# AI界面生成工具功能说明文档

## 项目概述
为"标桥·AI编标"产品的"解析招标信息"页面生成界面设计。

---

## 页面整体布局

采用三段式布局结构：

```
┌──────────────────────────────────────────────────────────────────────┐
│  顶部导航栏 (高度: 60px)                                                │
│  ├── 左侧：返回首页图标                                                  │
│  ├── 中间：步骤条 (创建项目 → 解析招标信息 → 生成正文)                      │
│  └── 右侧：字数余额显示 + 充值按钮                                        │
├──────────────────────────────────────────────────────────────────────┤
│  主内容区 (高度: 自适应剩余高度, padding: 24px)                          │
│  ┌───────────────────────┬─────────────────────────────────────────┐ │
│  │                       │                                         │ │
│  │  招标文件预览区        │        右侧信息展示区                    │ │
│  │  (宽度: 40%, 固定)     │        (宽度: 60%, 自适应)               │ │
│  │                       │  ┌───────────────────────────────────┐  │ │
│  │  - 显示上传的招标      │  │    项目信息展示区                  │  │ │
│  │    文件PDF预览        │  │    (标题 + 解析状态标签 + 编辑区)   │  │ │
│  │                       │  └───────────────────────────────────┘  │ │
│  │                       │  ┌───────────────────────────────────┐  │ │
│  │                       │  │    其他补充信息区                  │  │ │
│  │                       │  │    (系统提示 + 操作区)             │  │ │
│  │                       │  └───────────────────────────────────┘  │ │
│  │                       │                                         │ │
│  └───────────────────────┴─────────────────────────────────────────┘ │
├──────────────────────────────────────────────────────────────────────┤
│  底部操作栏 (高度: 72px, 居中显示按钮)                                   │
│  ├── 左侧：「上一步」按钮 (默认置灰，除非有前置操作)                      │
│  └── 右侧：「下一步」按钮 (流式输出期间置灰，结束后可点击)                │
└──────────────────────────────────────────────────────────────────────┘
```

---

## 模块一：顶部导航栏

### 样式规格
- 背景色：#FFFFFF
- 高度：60px
- 边框底部：1px solid #E5E5E5
- 左右内边距：24px

### 步骤条组件
- 三个步骤节点：①创建项目（已完成状态）、②解析招标信息（当前进行中状态）、③生成正文（待进行状态）
- 已完成状态：蓝色对勾图标 + 蓝色文字
- 进行中状态：蓝色圆形序号 + 蓝色文字 + 下方蓝色横线指示
- 待进行状态：灰色圆形序号 + 灰色文字
- 步骤间连接线：灰色虚线

---

## 模块二：招标文件预览区（左侧）

### 样式规格
- 宽度：占主内容区40%
- 背景：#F5F7FA
- 圆角：8px
- 内边距：20px
- 边框：1px solid #E8E8E8

### 内容展示
- 标题："招标文件" (字体14px, 加粗, 颜色#333)
- 文件预览：PDF文件预览组件，支持滚动查看
- 文件信息栏（底部）：显示文件名 + 文件大小 + 页数

---

## 模块三：项目信息展示区（右上）

### 样式规格
- 背景：#FFFFFF
- 圆角：8px
- 内边距：24px
- 边框：1px solid #E8E8E8
- 高度：自适应内容

### 标题区域
```
┌────────────────────────────────────────────────────────┐
│  项目信息展示    [解析中 / 解析完成]                      │
└────────────────────────────────────────────────────────┘
```
- 主标题："项目信息展示" (字体18px, 加粗, 颜色#1A1A1A)
- 状态标签：
  - 解析中：蓝色背景(#E6F7FF) + 蓝色文字(#1890FF) + 旋转loading图标
  - 解析完成：绿色背景(#F6FFED) + 绿色文字(#52C41A) + 对勾图标

### 信息模块列表（7个模块竖向排列）

每个信息模块采用卡片式布局：

```
┌────────────────────────────────────────────────────────┐
│  模块标题                                    [编辑图标]  │
├────────────────────────────────────────────────────────┤
│  内容区域                                               │
│  - 流式输出时：逐字显示，带光标闪烁效果                    │
│  - 流式结束后：可编辑文本区域                              │
│  - 解析为空时：显示"文件未提取到相关内容"占位文字           │
└────────────────────────────────────────────────────────┘
```

#### 各模块详细定义

| 序号 | 模块名称 | 模块标题 | 内容类型 | 编辑方式 | 占位提示 |
|------|----------|----------|----------|----------|----------|
| 1 | projectName | 项目名称 | 短文本 | 文本输入框 | 请输入项目名称 |
| 2 | bidDeadline | 开标时间 | 时间 | 时间选择器 | 请选择开标时间 |
| 3 | bidRequirements | 招标要求 | 长文本 | 多行文本框(4行) | 文件未提取到相关内容 |
| 4 | technicalCriteria | 技术评分标准 | 长文本 | 多行文本框(4行) | 文件未提取到相关内容 |
| 5 | bidDocCatalog | 投标文件组成目录 | 长文本 | 多行文本框(4行) | 文件未提取到相关内容 |
| 6 | projectList | 项目清单 | 长文本 | 多行文本框(4行) | 文件未提取到相关内容 |
| 7 | projectDrawings | 工程图纸 | 长文本 | 多行文本框(4行) | 文件未提取到相关内容 |

#### 模块卡片样式
- 模块标题：14px, 加粗, 颜色#333
- 编辑图标：铅笔图标，悬停显示"点击编辑"
- 内容区域背景：#FAFAFA
- 内容区域内边距：12px
- 内容区域圆角：4px
- 文本框边框：1px solid #D9D9D9，聚焦时变为#1890FF
- 模块间距：16px

#### 流式输出效果
- 文字逐字显示，间隔50ms
- 显示过程中有光标闪烁（|）跟随最后一个字
- 多个模块同时流式输出
- 已输出文字颜色：#333，正在输出的文字颜色：#1890FF

## 模块四：其他补充信息区（右下）

### 样式规格
- 背景：#FFFFFF
- 圆角：8px
- 内边距：24px
- 边框：1px solid #E8E8E8
- margin-top: 20px

### 标题
- "其他补充信息" (字体16px, 加粗, 颜色#1A1A1A)

### 子模块一：目录生成方式选择（条件显示）

**显示条件：** 投标文件组成目录 和 技术评分标准 两个模块均有解析内容

**显示时机：** 流式输出结束后立即显示

**组件样式：**
```
┌────────────────────────────────────────────────────────┐
│  📋 目录生成方式选择                                      │
│                                                         │
│  ℹ️ 检测到文件中同时提供技术评分标准及投标文件组成目录，    │
│     请选择目录生成方式                                   │
│                                                         │
│  ○ 融合投标文件组成目录/评分标准（默认选中）              │
│  ○ 仅按评分标准生成                                      │
│  ○ 仅按投标文件组成目录生成                              │
└────────────────────────────────────────────────────────┘
```

- 提示图标：蓝色信息图标
- 提示文字：14px, 颜色#666
- 单选框组：垂直排列，间距12px
- 默认选中第一项

### 子模块二：补充清单文件提示（条件显示）

**显示条件：** 第一步未上传清单文件

**组件样式：**
```
┌────────────────────────────────────────────────────────┐
│  📎 补充清单文件                                          │
│                                                         │
│  ⚠️ 当前未上传清单文件，建议补充上传                      │
│                                                         │
│  [选择文件]  清单文件.pdf                    [提交] [忽略]│
└────────────────────────────────────────────────────────┘
```

- 警告图标：橙色警告图标
- 提示文字：14px, 颜色#666
- 按钮组：「选择文件」主按钮 + 「提交」次按钮 + 「忽略」文字按钮

**状态流转：**
1. 初始状态：显示警告提示 + 「选择文件」按钮
2. 选择文件后：显示文件名 + 「提交」+「忽略」按钮
3. 点击提交后：显示"文件解析中" + loading动画
4. 解析完成后：提示消失，项目清单模块流式输出新内容

### 子模块三：补充图纸文件提示（条件显示）

**显示条件：** 第一步未上传工程图纸文件

**组件样式：** 与"补充清单文件"模块样式一致

## 模块五：底部操作栏

### 样式规格
- 背景：#FFFFFF
- 高度：72px
- 边框顶部：1px solid #E5E5E5
- 按钮居中排列，间距16px

### 按钮定义

| 按钮 | 默认状态 | 流式输出期间 | 点击行为 |
|------|----------|--------------|----------|
| 上一步 | 可点击 | 置灰不可点击 | 返回第一步 |
| 下一步 | 流式结束后可点击 | 置灰不可点击 | 进入第三步 |

**按钮样式：**
- 主按钮（下一步）：背景#1890FF, 文字白色, 圆角4px, 高度40px, 宽度120px
- 次按钮（上一步）：背景白色, 文字#666, 边框1px #D9D9D9, 圆角4px, 高度40px, 宽度120px
- 置灰状态：透明度0.5, 光标not-allowed

---

## 交互状态说明

### 状态一：页面初始化（刚进入第二步）

- 自动调用解析接口
- 「项目信息展示」标题旁显示"解析中"标签（蓝色 + loading图标）
- 7个信息模块同时开始流式输出
- 「上一步」「下一步」按钮均置灰
- 用户无法点击其他导航离开页面（可通过遮罩层实现）

### 状态二：流式输出进行中

- 信息模块内容逐字显示
- 光标闪烁跟随最后一个字
- 按钮保持置灰
- 导航保持锁定

### 状态三：流式输出完成

- 「项目信息展示」标题旁标签变为"解析完成"（绿色 + 对勾图标）
- 7个信息模块内容显示完整，变为可编辑状态
- 「下一步」按钮变为可点击
- 「上一步」按钮变为可点击
- 导航锁定解除
- 若满足条件，显示"目录生成方式选择"

### 状态四：补充文件解析中

- 点击补充上传的「提交」后
- 对应信息模块显示"解析中"
- 页面自动滚动定位至该模块
- 「下一步」按钮置灰
- 解析完成后流式输出新内容

**弹窗样式：**
```
┌────────────────────────────────────────┐
│  ⚠️ 解析失败              [关闭按钮]     │
├────────────────────────────────────────┤
│                                        │
│  招标文件解析失败，请检查文件格式        │
│  是否正确或稍后重试。                   │
│                                        │
│          [重新解析]                    │
│                                        │
└────────────────────────────────────────┘
```

- 宽度：400px
- 圆角：8px
- 背景遮罩：rgba(0,0,0,0.45)
- 关闭按钮：右上角X图标
- 「重新解析」按钮：主按钮样式
